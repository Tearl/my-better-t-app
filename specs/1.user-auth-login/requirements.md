# user-auth-login — 需求规格

## 概述

提供单页认证入口的「登录」能力与共享认证 UI 组件库，访客通过邮箱 + 密码登录并建立会话。

## 项目信息

- 项目名: my-better-t-app
- 架构类型: pnpm monorepo（Turborepo，apps/* + packages/*），后端以 AWS 云端资源承载

## 需求版本

| 日期       | 版本 | 说明     |
| ---------- | ---- | -------- |
| 2026-06-30 | v1   | 初始需求 |

## 用户故事

- 作为已注册用户，我想要用邮箱和密码登录，以便进入应用并保持会话。
- 作为访客，我想要在登录与注册之间一键切换，以便在同一页面完成不同操作。
- 作为用户，我想要在会话校验/提交过程中看到加载反馈，以便了解系统状态、避免重复提交。

## 功能需求

1. [F-001] 认证页采用单页双模式：顶部分段控件（`AuthTabs`）在「登录」「注册」间切换，底部文字入口（`AuthSwitchPrompt`）联动切换。
2. [F-002] 切换模式时重置表单、清空成功态、更新标题/副标题与校验规则；登录态标题「欢迎回来 / 登录您的账户继续使用」。
3. [F-003] 登录表单字段：电子邮箱、密码（密码字段支持显示/隐藏切换）。
4. [F-004] 登录校验（Zod，提交触发）：邮箱合法（「请输入有效的电子邮箱」）、密码至少 8 个字符（「密码至少需要 8 个字符」）。
5. [F-005] 校验通过调用 `authClient.signIn.email({ email, password })`；成功进入成功态（按钮「✓ 成功」）+ toast「登录成功」，失败 toast 显示后端错误信息。
6. [F-006] 会话加载态：`authClient.useSession()` 的 `isPending` 期间在认证背景中居中展示 `Loader`。
7. [F-007] 提交态：`isSubmitting` 期间按钮显示加载动画并禁用；`canSubmit` 为 false 时禁用按钮。
8. [F-008] 「忘记密码？」占位入口：点击 toast 提示「找回密码功能即将上线」，本期不实现真实流程。
9. [F-009] 共享认证 UI 组件库（`packages/ui`）：背景、玻璃拟态卡片、头部、分段控件、字段、提交按钮、分隔线、社交按钮、切换提示等可复用组件。

## 非功能需求

- 性能: 首屏认证卡片渲染无明显阻塞；会话校验加载态及时显示。
- 安全: 密码字段默认隐藏；所有非提交按钮使用 `type="button"` 避免误提交；不在前端持久化明文凭证。
- 兼容性: 响应式（移动端到桌面端），暗色主题（默认 `dark`）。
- 可访问性: 键盘可达所有交互元素，回车提交；字段具备可识别 `name`/`placeholder`。
- 规范: 通过项目 Ultracite/Biome 校验（`pnpm dlx ultracite fix`）。

## 验收标准

- [ ] [AC-001] 打开首页显示认证卡片，默认「登录」模式。
- [ ] [AC-002] 登录校验：邮箱非法、密码 < 8 时分别给出对应中文错误。
- [ ] [AC-003] 登录成功显示成功态并 toast「登录成功」，会话建立。
- [ ] [AC-004] 登录失败 toast 显示后端错误信息。
- [ ] [AC-005] 会话校验中显示 Loader；提交中按钮 loading 且禁用。
- [ ] [AC-006] 点击「忘记密码？」弹出「即将上线」提示且无报错。

## 依赖

- Better Auth（`better-auth/react`，邮箱 + 密码），baseURL 取 `VITE_SERVER_URL`。
- TanStack Router / React Form、Zod、sonner、lucide-react、Tailwind CSS v4。
- 内部 UI 包 `@my-better-t-app/ui`（本 feature 负责建立认证组件）。
