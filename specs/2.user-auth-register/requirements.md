# user-auth-register — 需求规格

## 概述

在单页认证卡片的「注册」模式下，访客通过用户名 + 邮箱 + 密码（含确认密码）完成注册并建立会话，并提供第三方登录占位入口。

## 项目信息

- 项目名: my-better-t-app
- 架构类型: pnpm monorepo（Turborepo，apps/* + packages/*），后端以 AWS 云端资源承载

## 需求版本

| 日期       | 版本 | 说明     |
| ---------- | ---- | -------- |
| 2026-06-30 | v1   | 初始需求 |

## 用户故事

- 作为新用户，我想要填写用户名、邮箱和密码注册账号，以便开始使用应用。
- 作为新用户，我想要在注册时确认密码一致，以便避免输错密码。
- 作为用户，我想要看到微信/QQ 等第三方登录入口，以便了解后续可选的登录方式。

## 功能需求

1. [F-001] 注册模式字段：用户名、电子邮箱、密码、确认密码（用户名/确认密码仅在注册模式显示）。
2. [F-002] 注册校验（Zod，提交触发）：用户名 ≥ 2 字符（「用户名至少需要 2 个字符」）、邮箱合法（「请输入有效的电子邮箱」）、密码 ≥ 8 字符（「密码至少需要 8 个字符」）。
3. [F-003] 确认密码必须与密码一致，否则报「两次输入的密码不一致」，错误归属到确认密码字段。
4. [F-004] 校验通过调用 `authClient.signUp.email({ name, email, password })`；成功进入成功态（按钮「✓ 成功」）+ toast「注册成功」，失败 toast 显示后端错误信息。
5. [F-005] 注册模式标题「创建账号 / 填写信息完成注册」；与登录模式通过 `AuthTabs` 和底部 `AuthSwitchPrompt`（「已有账号？去登录」）联动切换，切换时重置表单与成功态。
6. [F-006] 第三方登录占位：分隔线下方展示「微信登录」「QQ登录」按钮，点击 toast「{渠道} 登录即将上线」，本期不做真实对接。

## 非功能需求

- 性能: 模式切换与字段显隐无明显卡顿。
- 安全: 密码与确认密码字段默认隐藏；占位社交按钮 `type="button"`；不在前端持久化明文凭证。
- 兼容性: 响应式 + 暗色主题，与登录模式视觉一致。
- 可访问性: 字段具备可识别 `name`/`placeholder`，键盘可达、回车提交。
- 规范: 通过 Ultracite/Biome 校验。
